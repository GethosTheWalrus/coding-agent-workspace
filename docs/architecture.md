# Architecture Overview

This document provides a high‑level overview of the **Todo API** built with **FastAPI** and **SQLite**.

## Technology Stack
- **FastAPI** – modern, fast (high‑performance) web framework for building APIs with Python 3.9+.
- **Uvicorn** – ASGI server used to run the FastAPI application.
- **SQLAlchemy** – ORM for interacting with the SQLite database.
- **Pydantic (v2)** – data validation and settings management.
- **SQLite** – lightweight, file‑based relational database (no external service required).
- **pytest & httpx** – testing framework and async HTTP client for integration tests.

## Data Model
| Table | Columns |
|-------|---------|
| **todos** | `id` (PK, int, auto‑increment) <br> `title` (string, required) <br> `description` (string, optional) <br> `completed` (bool, default false) <br> `created_at` (datetime, auto) <br> `updated_at` (datetime, auto) |

## API Endpoints
| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/todos/` | List todos (supports `skip` & `limit` query params) |
| `GET` | `/todos/{todo_id}` | Retrieve a single todo |
| `POST` | `/todos/` | Create a new todo |
| `PUT` | `/todos/{todo_id}` | Update an existing todo |
| `DELETE` | `/todos/{todo_id}` | Delete a todo |

All endpoints return JSON payloads that conform to the Pydantic schemas defined in `app/schemas.py`.

## Startup Flow
1. **Uvicorn** starts the ASGI server and loads `app.main:app`.
2. FastAPI creates the `app` instance and registers the Todo router.
3. On the `startup` event, `Base.metadata.create_all()` ensures the SQLite tables exist.
4. Each request uses the `get_db` dependency to obtain a fresh SQLAlchemy session, which is closed automatically after the request.

## Extensibility
- **Authentication** – can be added via FastAPI dependencies and JWT tokens.
- **Pagination** – already supported via `skip` and `limit` query parameters; can be enhanced with total counts and links.
- **Migrations** – Alembic can be introduced for schema versioning.
- **Deployment** – containerize with Docker; add CI/CD pipelines for automated testing and deployment.
